Introduction & Purpose

This document defines the functional requirements for the Fashion Boutique Online Store project within the established project constraints. It describes what the system must do from the perspective of its users and the business, without specifying how it will be implemented technically.

This serves as the primary agreement between stakeholders and the development team for the system's capabilities. All requirements include a clear Definition of Done (DoD).

Project Vision & Objectives

Vision

To create a simple, elegant, and secure online storefront that extends the boutique's brand presence and enables 24/7 sales.

Primary Objective

Launch a Minimum Viable Product (MVP) within the severe time and budget constraints (40 core hours, zero monetary budget).

Success Criteria

  • A deployable system delivered within 40-hour timeline
  • Owner can independently manage core operations
  • Customers can complete purchases securely

Definition of Done (DoD)

For any functional requirement to be considered complete, all of the following conditions must be met:

1. Implemented

According to spec, using approved technology stack

2. Tested

Passes acceptance tests; no critical bugs (P1/P2)

3. Integrated

Works with existing components

4. Documented

Minimal necessary user/developer notes added

5. Reviewed & Merged

Code reviewed and merged to main branch

6. Accepted

Verified by Product Owner against FRD

Detailed Functional Requirements

Public Website (Frontend)

  • FR-01 Product Catalog Browsing

    DoD: List/Grid view, filter by category, sort by price. Loads acceptably on minimum hardware spec.

  • FR-02 Product Details Page

    DoD: Displays key attributes, images, stock status, functional "Add to Cart".

  • FR-03 Shopping Cart Management

    DoD: CRUD operations in cart, persisted for logged-in users, real-time subtotal.

  • FR-04 Secure Checkout Process

    DoD: 5-step flow with JWT session management. Inventory updates on success.

User Account & Security

  • FR-05 Secure User Registration & Login

    DoD: Secure password handling, JWT issuance and validation, session management.

  • FR-06 User Profile Management

    DoD: Users can manage name, email, and shipping addresses. Type safety enforced.

Administration Panel (Backend)

  • FR-08 Secure Admin Access Control

    DoD: Admin route protection via role-based middleware.

  • FR-09 Product Management (CRUD)

    DoD: Full product lifecycle management with image upload.

  • FR-11 Order Management & Fulfillment

    DoD: View all orders, update status (reflected in customer view).

Project Management Framework

Methodology

Agile Scrum adapted for a micro-team with extreme time constraints.

Sprint Structure

Four 10-hour "Micro-Sprints"
Total: (2.5h × 4 days) × 4 weeks = 40 hours

Daily Allocation

2.5 hours per day focused work
+ Possible 1 extra hour for critical issues only

Core Ceremonies

Ceremony Duration Purpose Frequency
Sprint Planning 15 min Prioritize backlog for next 10-hour block Per micro-sprint
Daily Stand-up 5-10 min Progress sync, blockers, daily focus Daily
Sprint Review 15 min Demonstrate completed work to Product Owner Per micro-sprint
Sprint Retrospective 10 min Identify one improvement for next sprint Per micro-sprint

Constraints & Limitations

Time Constraints

Total Development Time: 40 core hours (four 10-hour micro-sprints)

Daily Work: 2.5 hours × 4 days per week

Contingency: Possible 1 extra hour per day for critical issues only

Budget Constraints

Monetary Budget: $0 (Zero budget)

Implication: Must use free-tier services only for hosting, databases, and payment processing sandbox

Hardware Constraints

Optimal Spec: 32GB RAM, i7 Ultra 255H, 1TB SSD

Minimum Required: 16GB RAM, i5 CPU, 256GB SSD

GPU: 128MB (Standard display requirements)

Application must be designed to run functionally on minimum hardware

Team Skills Constraints

React: 70% proficiency
Node.js: 75% proficiency
MongoDB: 75% proficiency
SQL: 35% proficiency

Implication: Prioritize tasks using high-proficiency skills (>50%) to maximize efficiency

Security Requirements

Authentication

JWT-based authentication with clear role separation

Data Security

Environment secrets secured, password hashing, input sanitization

Code Practice

Adherence to OOP principles and type safety to reduce vulnerabilities

Key Technology Decisions (Based on Constraints)

  • Primary Stack: React/Node.js/MongoDB (highest team proficiency)
  • Avoid: Technologies with <50% team skill unless absolutely necessary
  • Security: Focus on JWT, input validation, role permissions (advanced infra like VPN/Vault out of scope)
